# Contingency Plans — Shared Backup Framework

> Backup activities and pivot rules for when plans change mid-class or a session needs replacing.

---

## Guest speaker backup rule

If the course has guest speakers scheduled (see `course_tracker.json`), maintain 1–2 fully designed backup class plans (timed, with activities) that can replace any guest speaker session with <24 hours notice. A backup plan is NOT "just do review" — it's a complete session with intent, energy arc, activities, and a timed plan.

---

## Pivot cheatsheet (mid-class improvisation)

| Situation | Pivot to |
|-----------|----------|
| Discussion wraps early (10+ min left) | Think-pair-share on a "what would you do?" scenario |
| Activity takes longer than planned | Cut second concept block, extend activity, recap key points |
| Energy drops visibly | 2-min stretch break, then think-pair-share or small group work |
| Students clearly didn't do the reading | Socratic walkthrough of the core framework; assign reading check next class |
| Tech failure (slides/video won't load) | Whiteboard the key framework, run discussion from memory |
| Guest speaker cancels day-of | Deploy backup class plan (see above) |
| Quiz takes much longer than expected | Shorten post-quiz discussion, push remaining content to next class |

### Activity template

When adding a backup activity, include:
- **Duration:** X minutes
- **Fits:** lecture / case / workshop / any
- **Materials:** what's needed (handout, whiteboard, none)
- **Instructions:** step-by-step for the instructor
- **Deliverable:** what students produce (if anything)
- **Debrief:** 2–3 wrap-up questions

## Content overflow protocol

When a class runs long and content gets pushed to the next session:
1. Note exactly what was deferred in the class log ("Content pushed to next class" section)
2. Add an action item to the tracker: "Integrate deferred content from Class N into Class N+1 prep"
3. Update `class_prep_reference.md` for the next class to include the deferred material
4. Do NOT try to "make up" skipped content by cramming — redesign the next class to integrate it naturally
