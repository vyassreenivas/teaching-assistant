# Weekly Rhythm — Shared Teaching Cadence

> This file defines the prep cadence philosophy that applies to every course. Course-specific timing (section gaps, day-of-week details) is populated by the generator.

---

## Core principle: stay a full week ahead

The goal is that when you walk into Class N, Class N+1 is already fully prepped. The post-class check-in after Class N triggers session design for Class N+2 (the one AFTER next). This means you're always working on two horizons:

- **Confirm horizon** (next class): should already be done, just review and lock
- **Design horizon** (class after next): this is where creative work happens

---

## Session design process (4 rounds)

When designing any class session, follow this sequence:

### Round 1: Intent
What should students walk out of this class knowing, feeling, or able to do? Write 2–3 bullet points. Everything else flows from this.

### Round 2: Energy arc
What's the emotional/cognitive trajectory? Common patterns:
- **Build**: hook → concept → apply → reflect (standard lecture-activity)
- **Case**: quiz → case discussion → framework → debrief (case class)
- **Workshop**: brief frame → extended hands-on → share-out (project work day)
- **Guest**: intro → speaker → Q&A → debrief (guest speaker)

### Round 3: Activities
What activities serve the intent and fit the energy arc? Draw from:
- The course playbook (if `../../knowledge/playbook.md` exists)
- Past class logs (what worked before on similar topics)
- The contingency backup bank

### Round 4: Timed plan
Map everything to the class time template. Account for transitions (1–2 min between segments). The timed plan becomes `Session Plan.md` in the class folder.

## Prep timing

| Deadline | What should be done |
|----------|-------------------|
| Post-class check-in | Debrief current class, trigger design for N+2 |
| Sunday 7 PM | Tuesday's class confirmed and locked |
| Wednesday 7 PM | Thursday's class confirmed and locked |

"Confirmed" means: slides done, activities prepped, materials ready, quiz built (if case class), Canvas announcement drafted. "Locked" means: no more changes unless something breaks.

---

## Grading rhythm

- Quiz grading: within 48 hours of the quiz (aim for same day if possible)
- Assignment grading: within 7 days (hard ceiling — SET Q1.7)
- Presentation grading: within 7 days
- Midterm grading: within 7 days

Sunday grading audit: check what's outstanding, flag anything approaching the 7-day limit.

---

## Midterm question bank

After every class, consider: "Did anything in today's material lend itself to a good exam question?" If yes, add it to `exam_bank.md` immediately. Don't wait until midterm week to write all questions at once.

---

## Start-of-conversation briefing

When the assistant opens a new conversation, it should proactively report:
1. Current date and where we are in the quarter (Class N of total)
2. What's due this week (from tracker)
3. Any overdue action items
4. Grading status (anything approaching 7-day limit)
5. Next class topic and prep status
